# Playroom Demo

## Описание проекта

**Playroom Demo** — это desktop-приложение на JavaFX 26 для управления игровой комнатой. Приложение предоставляет
функционал для бронирования игровых комнат, аренды игрового оборудования и управления настольными играми.

### Основные возможности

- **Бронирование комнат**: Почасовое бронирование 3 игровых комнат
- **Аренда оборудования**: Управление прокатом консолей, игр и геймпадов с почасовой оплатой
- **Инвентарь настольных игр**: Учёт и управление настольными играми
- **Система ролей**:
    - **ADMIN** — полный доступ ко всем операциям (создание, чтение, обновление, удаление)
    - **USER** — просмотр информации и возможность аренды/бронирования

## Технологический стек

- **Java 26** с использованием records и Lombok
- **JavaFX 26.0.1** (javafx.controls, javafx.fxml)
- **Gradle 9.4.0** — система сборки
- **SQLite** — база данных по умолчанию (можно переключить на H2 или PostgreSQL)
- **JDBC** — работа с базой данных без ORM
- **SnakeYAML** — конфигурация приложения
- **SLF4J + Logback** — логирование

## Архитектура

Приложение построено по паттерну **MVC** с дополнительным слоем Repository:

- **Models** (`model/`) — модели данных
- **Views** — FXML файлы для интерфейса
- **Controllers** (`controller/`) — контроллеры для обработки логики UI
- **Repositories** (`db/`) — слой доступа к данным
- **Services** (`service/`) — бизнес-логика

## Требования

- **Java 26** или выше
- **Gradle 9.4.0** (wrapper включен в проект)

## Установка и запуск

### 1. Клонирование репозитория

```bash
git clone <repository-url>
cd playroom-demo
```

### 2. Сборка проекта

```bash
./gradlew build
```

Для Windows:

```cmd
gradlew.bat build
```

### 3. Запуск приложения

```bash
./gradlew run
```

Для Windows:

```cmd
gradlew.bat run
```

## Конфигурация

Настройки приложения находятся в файлах:

- `src/main/resources/app.yml` — настройки подключения к базе данных
- `src/main/resources/users.yml` — учётные записи пользователей и их роли

### Пример конфигурации базы данных (app.yml)

```yaml
  url: jdbc:sqlite:sample.db
  user: ""
  password: ""
```

### Пример конфигурации пользователей (users.yml)

```yaml
users:
  - username: admin
    password: admin123
    role: ADMIN
  - username: user
    password: user123
    role: USER
```

## База данных

При первом запуске автоматически создаются следующие таблицы:

- `equipment` — оборудование (консоли, игры, геймпады)
- `room_booking` — бронирования комнат
- `tabletops` — настольные игры

### Загрузка тестовых данных

Для добавления данных в БД используйте Gradle-задачу (работает на macOS, Linux и Windows без установки sqlite3):

```bash
./gradlew loadTestData
```

Для Windows:

```cmd
gradlew.bat loadTestData
```

## Логирование

Логи приложения сохраняются в файл `playroom-demo.log` в корневой директории проекта.
